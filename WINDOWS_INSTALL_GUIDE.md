# Windowsユーザー向けインストール・利用ガイド

本ガイドは、WSL2 や Docker などの開発ツールがインストールされていない Windows 環境から、本シミュレーターアプリ（FFServer）を起動して利用するまでの手順を分かりやすく説明します。

---

## 1. 動作環境の確認（GPUの有無）

お使いの PC に NVIDIA 製のグラフィックボード（GeForce RTX 30シリーズ、40シリーズ、50シリーズなど）が搭載されているかどうかで、セットアップ手順の一部が異なります。

* **パターンA：RTX系 GPU 搭載 PC**（高速な計算が可能。推奨）
* **パターンB：GPU 非搭載 / RTX以外の PC**（CPUのみで動作。計算速度は遅くなりますが動作可能です）

自分の PC のグラフィックボードを確認するには：
1. キーボードの `Ctrl + Shift + Esc` を押して「タスク マネージャー」を開きます。
2. 「パフォーマンス」タブをクリックし、左側リストの下部にある「GPU」を選択します。
3. 右上に「NVIDIA GeForce RTX ...」と表示されていれば **パターンA**、それ以外（Intel, AMD 等）であれば **パターンB** になります。

---

## 2. インストール手順

以下のステップに従って、必要なソフトウェアを順にインストールしていきます。

### ステップ 1：NVIDIA グラフィックドライバの更新（パターンA：GPU搭載 PC のみ）
GPU の性能を最大限に引き出し、Docker から認識させるために最新のドライバをインストールします。
1. [NVIDIA公式ドライバダウンロードページ](https://www.nvidia.co.jp/Download/index.aspx?lang=jp) にアクセスします。
2. お使いのグラフィックボード（例: GeForce RTX 4070 等）を選択し、最新の「Game Ready ドライバ」または「Studio ドライバ」をダウンロードしてインストールします。
3. インストール完了後、PC を一度再起動します。

---

### ステップ 2：Docker Desktop のインストール（全員共通）
コンテナ仮想化ツールである「Docker Desktop」をインストールします。このツールをインストールする過程で、Windows 内で Linux を動かす仕組み（WSL2）も自動的にセットアップされます。

1. [Docker Desktop公式サイト](https://www.docker.com/products/docker-desktop/) にアクセスし、「Download for Windows」ボタンをクリックしてインストーラーをダウンロードします。
2. ダウンロードした `Docker Desktop Installer.exe` を実行します。
3. **重要：** インストール中の設定画面で **「Use WSL 2 instead of Hyper-V (recommended)」** に必ずチェックが入っていることを確認してください。
4. インストールが完了すると、Windows のサインアウトまたは再起動を求められますので、指示に従ってください。
5. 再起動後、Docker Desktop が起動し、チュートリアル画面が出れば完了です。（設定等はデフォルトのままで構いません）

---

### ステップ 3：Ubuntu（Linux）のインストール（全員共通）
シミュレーターの起動スクリプトを実行するための Linux 環境（Ubuntu）を Windows ストアから導入します。

1. Windows の「スタート」ボタンをクリックし、**「Microsoft Store」** アプリを開きます。
2. 検索バーに **「Ubuntu」** と入力し、検索結果から「Ubuntu 22.04 LTS」（または最新の「Ubuntu」）を選択して「入手」または「インストール」をクリックします。
3. インストール完了後、「開く」をクリックします。
4. 最初の一度だけ、Linux 用のユーザー名（username）とパスワード（password）の作成を求められます。画面の指示に従って半角英数字で入力してください。（※パスワード入力時、画面には文字が表示されませんが入力は受け付けられています）
5. ターミナル（黒い画面）にプロンプト（`username@...:~$`）が表示されれば、Linux 環境の準備は完了です。

---

### ステップ 4：プロジェクトのダウンロードと起動（全員共通）
Ubuntu ターミナルからシミュレーターアプリをダウンロードして起動します。

1. **Ubuntu ターミナル** を開きます（スタートメニューから「Ubuntu」を選択して起動できます）。
2. アプリのソースコードをダウンロードするために、以下のコマンドを入力して `Enter` キーを押します：
   ```bash
   git clone https://github.com/wak-shogo/FFServer.git
   ```
3. ダウンロードしたフォルダに移動します：
   ```bash
   cd FFServer
   ```

   > [!IMPORTANT]
   > **パターンA (GPU搭載) でGPUを正しく認識させるための設定**
   > - **Docker Desktopを使用する場合**：Docker Desktopの「Settings (歯車アイコン)」->「Resources」->「WSL integration」を開き、**「Enable integration with my default WSL distro」**（および「Ubuntu」のスイッチ）にチェックが入っていることを確認してください。
   > - **GPUが認識されない場合、またはDocker Desktopを使用せずUbuntu単体にDockerをインストールする場合**：
   >   プロジェクトフォルダに同梱されている以下の初期セットアップスクリプトを実行してください。WSL2のUbuntu環境に Docker と GPU認識用の **NVIDIA Container Toolkit** が自動的にインストール・設定されます。
   >   ```bash
   >   bash setup_wsl2.sh
   >   ```
   >   ※`setup_wsl2.sh` を実行した場合は、設定を反映するために Windows の PowerShell またはコマンドプロンプトで `wsl --shutdown` を実行し、WSL2を一度再起動してから次のビルド手順に進んでください。

4. アプリケーションを実行するためのコンテナイメージを構築（ビルド）します（数分かかります）：
   ```bash
   bash build_container.sh
   ```
5. ビルド完了後、アプリを起動します：
   ```bash
   bash start_all.sh
   ```
   * **パターンA（GPU搭載）の場合：** 自動的に GPU（CUDA）が認識され、`Started successfully with GPU support.` と表示されます。
   * **パターンB（GPU非搭載）の場合：** 自動的に CPU モードにフォールバックして起動し、`Started WITHOUT GPU support.` と警告が出ますがそのまま動作します。

これでセットアップと起動はすべて完了です！

## 3. シミュレーターアプリの使い方

起動が成功したら、Google Chrome などのブラウザを開き、アドレスバーに以下の URL を入力してアクセスしてください。

* **シミュレーター Web UI（GUI）**：[http://localhost:8511](http://localhost:8511)
* **Jupyter Lab（詳細解析用）**：[http://localhost:8888](http://localhost:8888)

### 各機能の利用手順

#### ① NPT シミュレーションを実行する（メイン画面）
加熱・冷却プロセスにおけるセルの体積や格子定数の変化を調べます。
1. **パラメータの設定** (サイドバー内):
   - **Start Temp / End Temp / Temp Step**: シミュレーションを行う温度範囲を設定します。（例: 10K から 1000K まで、5K 刻み）
   - **Magmom species tracking**: 磁気モーメントを追跡したい元素（例: `Co`）を入力して `+` ボタンを押す（または `Enter`）と、追跡用タグが追加されます。複数指定も可能です。
2. **ファイルのアップロード**:
   - `Select CIF Files` エリアに、結晶構造ファイル（`.cif`）をドラッグ＆ドロップするか、クリックして選択します。
3. **実行**:
   - 「Add NPT Jobs to Queue」をクリックすると、計算ジョブがキューに追加され、自動的にバックグラウンドでシミュレーションが始まります。
   - 実行中は、**「Live Monitoring」** グラフにセルの各軸の長さや体積の変化がリアルタイムで描画されます。

#### ② 構造最適化（Relaxation）を実行する
セルのエネルギーが最も安定する状態に原子配置を最適化します。
1. 上部のタブから **「Structure Optimizer」** を選択します。
2. `Select CIF File` エリアに CIF ファイルをドラッグ＆ドロップ（またはクリックして選択）します。
3. サイドバーの「Add OPT Job to Queue」をクリックすると、最適化計算が開始されます。

#### ③ 計算結果を確認・ダウンロードする
1. 上部のタブから **「Project Browser」** を選択します。
2. 左側のリストから実行したプロジェクト名（例: `20260626_...`）をクリックします。
3. **結果の可視化**:
   - 格子定数や体積の温度依存性プロット、動的結合長、RDF（動的動径分布関数）などの綺麗なグラフ画像がその場に表示されます。
4. **データのダウンロード**:
   - 計算結果の生データ（CSVファイルなど）や、最適化後の CIF ファイル、アニメーション用の軌跡データ（XYZファイル）を個別にダウンロードできます。
   - 「Create/Update ZIP Pack」ボタンを押すと、すべてのプロットや結果データをまとめた ZIP アーカイブが自動生成され、一括でダウンロードできます。

#### ④ 結晶構造を編集・加工する
結晶中の特定の原子を他の元素に置換したり、空孔（欠陥）を作成した CIF ファイルを新しく作成できます。
1. 上部のタブから **「CIF Structure Editor」** を選択します。
2. 「Operation Type」で **Substitution**（置換）または **Vacancy**（空孔）を選択します。
3. ターゲットとなる元素（例: `Bi`）、置換後の元素（例: `Mg`）、パーセンテージ（例: `10%`）を入力します。
4. 元となる CIF ファイルをドラッグ＆ドロップし、「Run CIF Processor」をクリックします。
5. 生成された新しい CIF ファイルをダウンロードして、そのままシミュレーションや最適化の入力ファイルとして利用できます。

## 4. 日常の運用と停止方法

### アプリの停止
Ubuntu ターミナルで `Ctrl + C` を押すか、以下のコマンドを実行するとコンテナが停止します。
```bash
docker stop my_calc_env
```

### 次回以降の起動方法
次回からは、Docker Desktop を起動した状態で、Ubuntu ターミナルを開いて以下のコマンドを実行するだけです。
```bash
cd FFServer
bash start_all.sh
```

### トラブルシューティング
* **ブラウザが開かない / 接続できない場合**：
  Ubuntu ターミナルで `docker exec my_calc_env supervisorctl status` を実行し、`api`, `worker`, `jupyter` がすべて `RUNNING` になっているか確認してください。もし停止している場合は、`bash start_all.sh` を再実行してください。
* **GPUが認識されなくなった場合**：
  WSL2 の一時的な不具合の場合があります。Windows 側の PowerShell またはコマンドプロンプトを開き、`wsl --shutdown` を実行して WSL を完全シャットダウンした後に、再度 Ubuntu ターミナルを開いて起動してください。
